# 데이터 증강 실험 로그: v2 (명암비 및 선명도 강화)
**날짜**: 2026-02-14  
**실험 환경설정**: `planned_v2.json`  
**대상 데이터셋**: `merged_dataset` (729장)  
**출력 경로**: `ai_research/augmentation_experiments/outputs/planned_v2`

## 1. 실험 목표
*   **드론 점검 환경 시뮬레이션**: 드론이 촬영하는 실제 인프라 점검 환경과 유사한 조건 형성.
    *   **거리**: 약 3m 내외
    *   **각도**: 정면 촬영 위주
    *   **초점**: 수직 이동 (벽면, 기둥 등의 균열 스캔)
*   **화질 개선 (Quality Enhancement)**: 조명이 좋지 않거나 초점이 흔들리는 상황에서 결함을 더 잘 식별하도록 개선.
    *   **그림자**: 그늘진 영역(Shadow)에 숨은 균열을 찾기 위해 국소 대비(CLAHE) 강화.
    *   **블러/흔들림**: 드론의 진동 및 초점 불안정성을 보정하고 학습.

## 2. 증강 파이프라인 구성 (Augmentation Pipeline)
`Albumentations` 라이브러리를 사용하여 다음 변환들을 적용했습니다:

| 변환 기법 (Transform) | 주요 파라미터 | 적용 이유 (Reason) |
| :--- | :--- | :--- |
| **Motion/GaussianBlur** (OneOf) | `p=0.3`, `blur_limit=7` | 비행 중 발생하는 카메라 흔들림 및 초점 오류를 학습하여 강건성 확보. |
| **ISONoise** | `p=0.3`, `intensity=[0.1, 0.5]` | 저조도 환경이나 고감도(ISO) 촬영 시 발생하는 센서 노이즈 시뮬레이션. |
| **CLAHE** | `p=0.5`, `clip_limit=2.0` | **(핵심)** 어두운 영역의 대비를 높여 그림자 속에 숨은 미세 균열을 강조. |
| **Sharpen** | `p=0.5`, `alpha=[0.2, 0.5]` | **(핵심)** 흐릿한 콘크리트 질감 위에서 균열의 가장자리(Edge)를 선명하게 분리. |
| **ShiftScaleRotate** | `shift_y=0.1`, `rotate=5` | 수직 방향의 미세한 위치 이동과 안정적인 비행(±5도 회전) 상황 반영. |
| **Resize** | `512x512` | 엣지 디바이스(Jetson 등)에서의 추론 속도 최적화를 위한 입력 크기 조정. |

## 3. 데이터 전처리 및 해결된 문제
### A. 클래스 불일치 해결 (Class Unification)
*   **문제점**: `merged_dataset` 병합 과정에서 팀원 간 오타로 인해 두 개의 클래스 ID가 혼재됨.
    *   `ID 0`: "defect" (479장) - 정상
    *   `ID 1`: "defecct" (455장) - **오타**
*   **해결 방안**: `augmentation_template.py` 스크립트 내에 강제 매핑 로직 추가.
    *   **조치**: 입력되는 어노테이션의 ID가 무엇이든 무조건 **Class ID 0 (Crack)**으로 변환하여 저장.
    *   **결과**: 생성된 모든 데이터의 라벨이 `0`으로 통일됨.

### B. 포맷 변환 (Format Conversion)
*   **입력**: COCO JSON (단일 파일)
*   **출력**: YOLO `.txt` (이미지별 개별 파일)
    *   좌표계 변환: `[x_min, y_min, w, h]` (픽셀 단위) → `[x_center, y_center, w, h]` (0~1 정규화).
