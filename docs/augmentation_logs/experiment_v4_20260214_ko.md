# Augmentation Experiment V4 Report (2026-02-14)

## 1. 실험 개요 (Overview)
*   **실험명**: `planned_v4_ratio_1_2`
*   **목적**: `planned_v3` (1:1 혼합) 모델의 성능을 넘어설 수 있는지 확인. **증강 데이터의 비율을 2배로 늘렸을 때(1:2)**의 효과 검증.
*   **날짜**: 2026-02-14
*   **담당**: Choi (AI Engineer)

## 2. 가설 및 전략 (Hypothesis & Strategy)
**"증강 데이터의 비율을 높이면(1:2), 모델이 더 다양한 변형에 노출되어 일반화 성능(Generalization)이 향상될 것인가?"**

*   **비교 대상**: `planned_v3` (1:1 비율, mAP50: 0.797)
*   **데이터 구성**:
    1.  **Original (729장)**: 1배수 (Baseline)
    2.  **Augmented (1,458장)**: 2배수 (`num_aug_repeats=2`)
    *   **총 데이터셋**: 약 2,187장

## 3. 변경 사항 (Changes)
### Configuration (`planned_v4.json`)
*   `include_originals`: `true`
*   `num_aug_repeats`: **2** (New Param) - 원본 1장당 증강 이미지 2장 생성.

### Validation Set
*   전체 데이터(2,187장)를 무작위로 섞은 후 20% 분할.
*   Validation Set 내부 비율도 통계적으로 **1:2 (Original:Augmented)**를 따름.

## 4. 예상 결과 (Expected Outcome)
*   **Positive**: 다양한 변형을 더 많이 학습하여 `mAP50-95`(정밀도)나 `Recall`이 소폭 상승할 수 있음.
*   **Negative**: 원본 비율이 상대적으로 줄어들어(33%), `v3`(50%)보다 원본 데이터에 대한 `Fitness`가 떨어질 수도 있음.

## 5. 실행 계획 (Action Plan)
1.  `augmentation_template.py`로 데이터셋 생성 (완료)
2.  `planned_v4.zip`을 Colab에 업로드하여 학습 (진행 예정)
3.  학습 완료 후 `v3` 모델과 성능 지표 비교

## 6. 실험 결과 비교 (Results Comparison: v3 vs v4)

| Model | Ratio (Orig:Aug) | Precision | Recall | mAP50 | mAP50-95 | 비고 |
| :--- | :---: | :---: | :---: | :---: | :---: | :--- |
| **v3** | 1:1 | 0.783 | **0.769** | 0.797 | **0.595** | **Best Recall** (균형 잡힘) |
| **v4** | 1:2 | **0.851** | 0.743 | **0.828** | 0.566 | **Best Precision** (오탐 최소화) |

### 7. 상세 분석 (Detailed Analysis)
1.  **Precision 상승 (0.78 -> 0.85)**: 증강 데이터 비율을 높이니(2배), 모델이 노이즈나 복잡한 배경 속에서도 균열이 아닌 것을 걸러내는 능력이 탁월해짐.
2.  **Recall 소폭 하락 (0.77 -> 0.74)**: 원본 데이터의 비중이 상대적으로 줄어들어(50% -> 33%), 평범하고 쉬운 균열을 놓치는 경우가 조금 늘어남.
3.  **mAP50 최고 기록 (0.828)**: 정밀도가 크게 오르면서 전체적인 탐지 점수는 0.83에 도달함. YOLO Nano 모델치고는 매우 훌륭한 수치.
4.  **mAP50-95 (0.566)**: v3(0.595)보다 소폭 낮음. 이는 **박스를 아주 정밀하게(Tight)** 그리는 능력은 v3가 조금 더 낫다는 의미. 하지만 0.5가 넘는 수치는 실사용에 충분함.

### 8. 결론 (Conclusion)
*   **v3 (1:1)**: **안전 제일**. 놓치는 균열을 최소화하고 싶다면 v3가 유리함.
*   **v4 (1:2)**: **효율 중심**. 오탐(False Alarm)으로 인한 피로도를 줄이고 싶다면 v4가 유리함.

**최종 제언**: 두 모델 모두 우수하나, **균열 탐지의 목적(안전 진단)상 Recall이 더 높은 `v3`를 Base 모델로 유지**하되, 현장 테스트에서 오탐이 너무 많다면 `v4`로 교체하는 것을 추천함.
