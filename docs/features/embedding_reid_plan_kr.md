# 중복 균열 방지를 위한 임베딩 기반 재식별(Re-Identification) 도입 계획서

## 1. 개요 및 목적
현재 Wall-E 프로젝트에서는 **YOLO 자체 내장 트래킹 알고리즘(새로운 객체가 탐지될 때마다 고유 ID 생성)**을 사용하여 중복 이미지 저장을 방지하고자 했습니다. 하지만 드론 비행의 특성상 지속적인 이동, 심한 카메라 흔들림, 앵글의 변화 등으로 인해 객체 ID가 쉽게 끊기고, 이전에 이미 탐지되어 저장된 균열이 다시 새로운 ID를 발급받으며 중복 저장되는 치명적인 한계점이 발견되었습니다.
이러한 트래킹 ID 방식의 한계와 기존 딥러닝 모델 재학습(Catastrophic Forgetting, 실시간 딜레이 유발)의 단점을 모두 극복하기 위해, 가장 가볍고 효율적인 대안인 **임베딩(특징 벡터) 기반 유사도 비교 방식(Re-Identification)**으로 전환하여 실시간 균열 이미지 중복 필터링을 완벽히 구현하고자 합니다.

---

## 2. 작동 방식 및 아키텍처

### 2.1 처리 파이프라인 (Pipeline)
1. **탐지 및 크롭 (Detection & Crop)**
   - 메인 스트림에서 **YOLOv11n** (또는 YOLO11n) 모델이 새로운 균열(Bounding Box)을 탐지합니다.
   - 원본 프레임에서 해당 Bounding Box 영역만 잘라냅니다(Crop).
2. **특징 추출 (Feature Extraction)**
   - 잘라낸 이미지를 매우 가벼운 딥러닝 모델(Feature Extractor - 예: MobileNetV3, ResNet18 등)에 통과시킵니다.
   - 이 과정을 거치면 이미지가 고유한 숫자 배열(1차원 벡터, 예를 들어 크기가 512인 벡터)인 **'임베딩(Embedding)'**으로 변환됩니다.
3. **벡터 저장 (Vector Storage)**
   - 새롭게 추출된 임베딩은 메모리(혹은 캐시 서버) 상의 '최근 탐지 목록 리스트'나 빠른 벡터 매칭 로직으로 넘어갑니다.
4. **유사도 비교 (Cosine Similarity)**
   - 새로운 임베딩과, 최근 1~2분(혹은 현재 미션 내내) 내에 저장되었던 기존 균열들의 임베딩들 간의 **코사인 유사도(Cosine Similarity)**를 수학적으로 계산합니다.
5. **필터링 결정 (Decision)**
   - 만약 두 임베딩 간의 유사도가 설정한 임계값(Threshold, 예: 85% 이상)을 초과한다면, "이미 찍힌 균열과 동일하다"고 판단하고 DB 저장 및 알림을 **스킵**합니다.
   - 일치하는 결과가 없다면 "새로운 균열"로 판단하고 갤러리에 저장하며 임베딩 풀에 새로 추가합니다.

## 3. 도입 시 얻을 수 있는 장점

### 3.1 ⚡ 극적인 실시간성 확보 (Zero Bottleneck)
- **가벼운 연산량:** 모델 전체의 가중치를 지속 업데이트해야 하던 '재학습 방식'과 달리, 무거운 역전파(Backpropagation) 과정이 없습니다.
- **초고속 유사도 계산:** 추출된 임베딩 1차원 배열끼리의 단순 내적 연산(코사인 유사도)이므로 Python 등에서 밀리초(ms) 단위로 완료됩니다. 30fps 환경의 부하를 최소화합니다.

### 3.2 🧠 이전 데이터 삭제 방지 (안정성)
- 기존 학습 방식의 치명적 오류였던 파국적 망각(Catastrophic Forgetting) 문제가 원천 차단됩니다.
- 새로운 특징을 추가할 때 기존 모델 가중치에 영향을 주는 방식이 아니기 때문에, 이전에 인식했던 균열에 대한 인식 로직이 전혀 망가지지 않습니다.
- 한 번 "균열이다"라고 분류했던 기준은 그대로 영구 보존됩니다.

### 3.3 💡 원근감 및 회전/흔들림에 대한 내성 (Robustness)
- 단순 픽셀 비교(OpenCV SSIM 등)나 해시값 매칭 수단들의 경우 카메라가 조금만 돌아가도 완전히 다른 이미지로 인식합니다.
- 그러나 ResNet이나 MobileNet과 같은 딥러닝 특징 추출기는 이미지의 '의미적 특징점(Semantic Features)'을 뽑아내기 때문에, **드론이 우측 대각선으로 조금 이동해서 찍건 줌을 조금 당겨서 찍건 동일한 균열임**을 강력하게 판별할 수 있습니다.

### 3.4 🖥️ 하드웨어/메모리(OOM) 리소스 최적화
- **VRAM 점유 최소화:** 백그라운드 학습 모델을 상시 올리고 있을 필요가 없고, 가벼운 Feature Extractor 하나만 추가로 GPU(또는 추론 속도에 따라 CPU)에 띄워두면 되므로 리소스 부담이 매우 낮습니다.

## 4. 실제 구현 및 튜닝 결과 (Implementation & Tuning)

본 계획에 따라 `MobileNetV3-small` 기반의 Re-ID 파이프라인이 성공적으로 백엔드(`reid_manager.py`)에 통합되었습니다.

1. **Lightweight Feature Extractor 로드**
   - 인터넷 연결이 없는 오프라인(폐쇄망) 환경에서도 작동하도록, 캐시 메모리가 아닌 로컬 폴더(`/models/mobilenet_v3_small.pth`)에서 가중치를 직접 읽어오도록 아키텍처를 구성했습니다.
2. **StreamManager 통합 및 튜닝 파라미터 적용**
   - 실시간 스트리밍 중 드론의 상하좌우 흔들림과 화각 변화에 더 강인하게 대응하기 위해 튜닝을 거쳤습니다.
   - **Crop Margin (10%):** YOLO가 탐지한 앙상한 박스에만 의존하지 않고, 주변 시멘트나 벽돌 문양 등의 컨텍스트 영역을 상하좌우 10%씩 더 잘라내어(Crop) 임베딩 모델에 먹입니다. 이를 통해 앵글이 틀어져도 동일한 위치임을 훨씬 잘 찾아냅니다.
   - **Threshold (0.80):** 유사도 커트라인을 0.85에서 **0.80**으로 하향 조정하여, 형태가 다소 일그러진 중복 균열도 놓치지 않고 쳐내도록(Drop) 방어력을 높였습니다.
3. **리스트 자동 비움 기능 (TTL)**
   - 무한 메모리 증식을 막기 위해, `deque(maxlen=50)` 자료구조를 사용하여 미션 당 가장 최근에 식별된 최대 50개의 균열 임베딩만 캐싱하도록 최적화되었습니다.
