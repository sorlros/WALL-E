# 창문(Window) 탐지 모델 실행 구조 및 최적화 아키텍처

## 1. 개요
본 문서는 Wall-E 프로젝트에서 드론의 실시간 스트리밍 중 건물의 창문을 탐지하고 사생활을 보호(블러/마스킹)하는 기능이 백엔드(MacBook)와 프론트엔드(Flutter)에서 어떻게 분산되어 실행되는지, 그리고 서버의 성능 부하를 최소화하기 위해 도입된 **비동기 메타데이터 스트리밍(Asynchronous Metadata Streaming)** 기법에 대하여 설명합니다.

---

## 2. 기존 방식 구조와 문제점
과거 일반적인 객체 탐지 + 스트리밍 구조는 다음과 같이 작동했습니다.

1. **카메라 입력:** 백엔드가 30FPS의 영상을 수신합니다.
2. **AI 추론 시계열 병목:** 한 프레임당 `균열 감지 모델` ➔ `창문 감지 모델` ➔ `코사인 유사도 (Re-ID) 검사`를 직렬로 모두 수행합니다.
3. **오버레이 렌더링 (가장 큰 부하):** 백엔드 소스코드(OpenCV)가 영상 프레임 픽셀 위에 직접 빨간색 바운딩 박스를 그리고, 창문 위치 픽셀을 직접 블러 처리(Gaussian Blur)합니다.
4. **재인코딩:** 이렇게 변경된 무거운 화상 데이터를 다시 비디오 파일 포맷(MJPEG)으로 인코딩하여 프론트엔드(Flutter)로 전송합니다.

**🚨 결과:**
이 방식은 백엔드 기기(MacBook)의 CPU/GPU 점유율을 100%에 도달하게 만들며, 결과적으로 1초에 5~10프레임밖에 전송하지 못해 영상이 심하게 끊기는 현상(Frame Drop)을 유발합니다.

---

## 3. 도입된 최적화 아키텍처 (비동기 BBox 전송)

현재 `stream_manager_ai_only.py`에 적용된 방식은 서버의 영상 조작(Rendering)을 완전히 없애고 프론트엔드(Client)에 렌더링을 위임하는 방식입니다.

### 3.1 백엔드(MacBook)의 역할
- **비디오 패스스루 (Pass-Through):** 백엔드는 들어오는 영상 그대로를 어떠한 그림(박스)도 그리지 않고 즉각적으로 프론트엔드에 송출합니다. (재인코딩 부하 0%)
- **프레임 스킵 (Frame Skip) 추론:** 창문은 건물에 고정되어 있어 드론 화면 내에서 위치가 천천히 변합니다. 따라서 30프레임 내내 창문을 찾을 필요가 없이, **3프레임당 1번(약 10FPS)**만 창문 모델(`window-model.pt`)을 듬성듬성 동작시킵니다.
- **메타데이터 비동기 전송:** 백엔드는 창문을 발견하면 영상을 건드리는 대신, 해당 창문의 정규화된 좌표값(`bbox_norm`) 정보만 가벼운 텍스트(JSON)로 WebSocket을 통해 프론트엔드에 별도로 전송합니다.

### 3.2 프론트엔드(Flutter)의 역할
- **원본 영상 재생:** `Mjpeg` 위젯을 통해 백엔드가 쏘아주는 깨끗하고 지연 없는 30FPS 원본 영상을 그대로 재생합니다.
- **오버레이 렌더링 (UI 덧그리기):** WebSocket으로 `type: "window"` 패킷이 도착하면, Flutter의 `CustomPainter`(`WindowBBoxPainter`)가 돌아가고 있는 동영상 층(Layer) 바로 위에 투명한 유리판을 댑니다.
- **사생활 보호 UI 적용:** 전달받은 상대 좌표 위치에 맞춰, 프론트엔드 디바이스(스마트폰/태블릿)의 GPU를 활용하여 반투명한 검은색 박스(`Colors.black.withOpacity(0.8)`)와 '사생활 보호 영역' 글씨를 실시간으로 덧그려 창문을 가려버립니다.

---

## 4. 구조적 장점 (Optimization Benefits)

1. **MacBook 과부하(발열) 방지:**
   - 렌더링을 완전히 클라이언트에 떠넘김으로써, 백엔드는 모델 3개(Crack, Window, Re-ID)를 동시에 구동하더라도 여력이 생기게 됩니다.
2. **비디오 지연(Latency) 완벽 해결:**
   - AI 분석이 무거워져서 0.1초 늦게 결괏값이 도출되더라도, 사용자가 보고 있는 뒷배경 드론 원본 영상은 끊기지 않고 부드럽게 재생됩니다.
3. **유연한 UI 수정:**
   - 이제 박스 색상, 블러 강도, 텍스트 디자인 등을 변경하고 싶을 때 무겁고 위험한 백엔드 AI 코드를 뜯어고칠 필요가 없습니다. Flutter UI 코드만 변경하면 즉시 반영됩니다.
4. **데이터 통신량 절약:**
   - 백엔드는 무거운 비디오에 그림을 입혀 보내는 대신, 가벼운 JSON 텍스트 몇 줄만 WebSocket으로 쏘므로 네트워크 병목이 대폭완화됩니다.
